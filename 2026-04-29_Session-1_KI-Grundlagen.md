## Themenblock 1 — Die drei Rollen von KI

### Marco's Einteilung:

**Rolle 1: Sparringspartner**
ChatGPT, Claude, Gemini, Perplexity, Grok, Notebook LM, Copilot — alle sind Sparringspartner.
- Gut für: Fragen, Texte, Ideen, Erklärungen, E-Mail-Antworten
- „Der weiß relativ viel — nicht die Details über euch, aber sehr viel Allgemeinwissen."
- Marcos Zeitanteil im Gespräch mit KI: **~70 %**

**Rolle 2: ITler / Programmierer**
KI schreibt Code, baut Skripte, verwaltet Netzwerke — auf Anweisung in natürlicher Sprache.
- „Nach 5–6 Wochen werdet ihr selber Dinge bauen, die ich euch auf der Messe gezeigt habe."
- Zeitanteil aller Marcos KIs (quantitativ): **~60 %** (weil KIs schlafen nicht, laufen durch)

**Rolle 3: Automatisierer**
KI baut und betreibt Fließbänder für Prozesse, die immer gleich ablaufen.

### Kernunterschied KI vs. Automatisierung:

| | Automatisierung | Künstliche Intelligenz |
|---|---|---|
| Verhalten | Deterministisch — immer gleich | Intelligent — findet Wege, löst Probleme |
| Beispiel | Lichtschalter → Licht an | E-Mail lesen und Person zuordnen |
| Einsatz | Wenn der Prozess immer gleich ist | Wenn Entscheidungen gefragt sind |

> „KI und Automatisierung sind zwei verschiedene Welten. KI kann Automatisierungen bauen. Aber innerhalb einer Automatisierung, wo Entscheidungen nötig sind, kommt dann ein Stück KI zum Einsatz."

**Marco's eigenes Beispiel:**
Dokumenten-Management: E-Mails kommen rein → Automatismus liest Text → KI ordnet Personen zu (braucht echte Intelligenz, keine stumpfe Wenn-Dann-Regel).

### ✅ Learning
- Die meisten KMUs nutzen KI nur als Sparringspartner (Rolle 1) — das ist „2024"
- Das echte Potenzial liegt in Rolle 2 + 3
- Faktor 10 mehr Anwendungsbereiche in Rolle 2+3 gegenüber Rolle 1

## Themenblock 2 — Was ist eine KI? (Grundlagen)

### Janines Definition (vom Coach als perfekt bewertet):
> „KI ist eine riesige Datenbank, die Informationen miteinander in Verbindung setzt und versucht herauszufinden, was die wahrscheinlichste [Fortsetzung] ist."

### Marco's Erklärung — Das Gehirn der KI

**Schritt 1: Die Tabelle (das Modell)**
Stell dir eine riesige Excel-Tabelle vor:
- Alle Wörter einer Sprache stehen in den Zeilen und Spalten
- In den Zellen steht: Wie häufig kommen diese zwei Wörter gemeinsam in der Literatur vor?

Beispiele:
- „Himmel" + „Blau" → 318 gemeinsame Vorkommen
- „Finanzamt" + „glücklich" → nur 7 gemeinsame Vorkommen

**Schritt 2: Das Training**
- Eine Trainingsmaschine durchsucht alle Texte der Welt (Internet, Bücher, Twitter, Blogs, Gerichtsurteile...)
- Sie zählt Wort-Kookkurrenzen und befüllt die Tabelle mit Häufigkeiten
- Das Ergebnis: Ein mathematisches Abbild unserer Welt in Wortkorrelationen

**Schritt 3: Der Transformer (GPT)**
GPT = Generative Pre-trained Transformer
- Nimmt einen Prompt entgegen
- Geht in die Tabelle (das Gehirn)
- Fragt: Was ist das **nächstwahrscheinlichste Wort?**
- Gibt genau **ein Token** zurück
- Dann: neuer Prompt = alter Prompt + das Token → wieder ein Token → ...
- So entsteht Wort für Wort eine Antwort

**KI in drei Worten:**
> **„Next Word Prediction"**

### Analogie Mensch ↔ KI

| Mensch | KI |
|---|---|
| Lernt durch 5 Sinneskanäle | Lernt nur durch Texte (Wörter/Token) |
| Korrelationen aus Erfahrungen | Korrelationen aus Häufigkeiten |
| Frontallappen filtert Antworten | Anbieter-System-Prompt filtert |
| Kausalketten als Erfahrung | Gewichtungstabelle als Wissen |

### ✅ Learning
- KI ist keine Magie — sie ist Wahrscheinlichkeitsrechnung auf einer Wort-Korrelationstabelle
- Jedes Wort in einer KI-Antwort kostet einen Transformer-Durchlauf
- Das Grundprinzip erklärt, warum KI manchmal seltsame Fehler macht: Sie folgt immer der höchsten Wahrscheinlichkeit, nicht der Wahrheit

## Themenblock 3 — Thinking / Reasoning Models

### Standard-Modus:
Prompt rein → sofort Transformer → Wort raus → Wort raus → ...
Kein Nachdenken, einfach losfeuern.

### Thinking-Modus:
KI macht erst „stille" Transformer-Durchläufe (nicht sichtbar), prüft eigene Antwort, startet ggf. neuen Gedankengang — dann erst spricht sie.

> „Wie ein Mensch, der kurz die Luft anhält und nachdenkt, bevor er antwortet — statt sofort loszureden."

Technisch gesehen: Die KI macht Next-Word-Prediction mit geschlossenem Mund — produziert Wörter, aber zeigt sie nicht. Erst wenn die KI mit dem Ergebnis zufrieden ist, gibt sie die Antwort aus.

**Sichtbar als:** Zusammenfassung der Gedankengänge (einige hundert Wörter, aber intern waren es 500–600+)

**Einstellungsmöglichkeiten (ChatGPT Pro):**
- Instant → sofort antworten
- Normal → kurz denken
- Intensiv → viel nachdenken (Bewerbungsgespräch-Modus)

**Synonyme:** Thinking = Reasoning = Extended Thinking = Effort → alles dasselbe

### ✅ Learning
- Bei schlechten Antworten: zuerst Reasoning-Aufwand erhöhen
- Für komplexe Aufgaben (viele Bedingungen, Präzisionsanforderung) lohnt Thinking-Modus sehr
- Hausaufgabe der nächsten 7 Tage: Wenn Antwort nicht gut genug → Reasoning hochstellen, beobachten

## Themenblock 4 — Tools & Werkzeuge: Der Harness

### Die zentrale Frage (Marvins Frage):
> „Woher weiß die KI, dass sie Werkzeuge benutzen darf und wie sie sie anwenden soll?"

### Antwort: System Prompts

Jedes Mal, wenn du etwas an die KI schreibst, werden unsichtbar **3 Arten von Prompts vorangestellt**:

**1. Anbieter-System-Prompt (immer dabei, vom Anbieter geschrieben)**
- ~6.000 Token (entspricht ~15 Minuten Sprachtext)
- Inhalt: „Du bist ein hilfreicher Assistent, du beleidigst nicht, du verhältst dich anständig..."
- Ohne diesen Prompt wäre die KI brutal ehrlich — würde bei einfachen Fragen beleidigen

**Warum ist KI ohne System Prompt unhöflich?**
> „Die KI beschreibt die Welt, wie sie ist — nämlich ehrlich. Wenn du nach Astrophysik fragst und die KI mehr weiß als du, sagt sie dir das direkt. Höflichkeit ist eine kulturelle Schicht, die erst durch System-Prompting reinkommt."

**2. Harness-System-Prompt (von der Plattform/Umgebung)**
- Erklärt der KI: In welcher Umgebung läufst du? Welche Werkzeuge hast du?
- Beispiel: „Du kannst ein Recherche-Tool aufrufen. Dafür schreib `tool_use: research`, gefolgt von Suchbegriffen."

**3. Customized System Prompt (selbst schreibbar)**
- Eigene Verhaltensanweisungen für die KI
- Beispiel: „Du sprichst mit Handwerkskunden. Gib keine Preise raus. Leite bei Kaufinteresse weiter."
- Das ist die „Einarbeitung eines neuen Mitarbeiters"

### Der Harness = Iron Man Anzug

> „Ein Modell steckt in einem Anzug — und der Anzug stellt Werkzeuge zur Verfügung. Das Modell ruft Werkzeuge auf, indem es bestimmte Codewörter schreibt. Der Anzug hört mit und löst das Tool aus."

Technisch:
- KI schreibt im Transformer-Lauf: `tool_use: research`, Keywords: `Marvin Hensel`
- Der Harness (ein normales Programm, kein KI) erkennt das Codewort → löst Recherche aus
- Ergebnis kommt zurück → KI verarbeitet es weiter

### Bekannte Tools in heutigen Harnessen:
- Internetrecherche
- Code-Ausführung / Programmierung
- Bildgenerierung
- Browser Use / Computer Use → Mausklicks, Screenshots, Portal-Logins (Tobias' Frage)

> Computer Use: KI macht Screenshot, erkennt wo sie ist, klickt weiter → kann sich in Ausschreibungsportale einloggen und Daten rausholen. Nicht datenschutzkonform mit echten Daten — aber mit Fake-Daten oder eigenen Portalen möglich.

### ✅ Learning
- ChatGPT.com = Harness mit ~7 Tools → sehr limitiert
- Ab Session 2: Ein Harness mit weit mehr Tools + Möglichkeit, eigene Tools zu bauen
- Ziel: KI kommt in die „echte Welt" — kann Dinge tun, nicht nur antworten

## Themenblock 5 — Agents

### Marcos Definition:
> „Wenn eine KI in der Lage ist, eine Werkzeuglandschaft zu begreifen und sinnvoll einzusetzen — dann sprechen wir von einem Agenten."

- Agent = LLM + Tool Usage (sinnvoll kombiniert)
- ChatGPT ist genau genommen auch ein Agent — aber mit sehr begrenzter Tool-Palette
- Ziel: Ein Harness mit „unbegrenzter" Tool-Palette + Fähigkeit, eigene Tools zu bauen

**Extrembeispiel Boeing 747:**
> „Wenn wir eine Boeing 747 von einer KI fliegen lassen wollen, erschaffen wir genug kleine Werkzeuge für alle Kontrollmechanismen — die KI koordiniert dann die Tools intelligent."

**Was das bedeutet für Betriebe:**
- Buchhaltung: KI versteht den Buchungssatz, holt die Zahlen, führt ihn aus
- Ausschreibungsportal: KI loggt sich ein, liest Ausschreibungen, extrahiert Infos
- Dokumentenmanagement: KI liest, sortiert, ordnet zu — ohne Mensch

### ✅ Learning
- Die wahre Stärke von KI liegt nicht im Gespräch, sondern in Intelligenz + Werkzeug-Kombination
- KI ohne Werkzeuge ist wie ein kluger Mensch ohne Arme

## Themenblock 6 — Modelle, Feintuning und Prompting

### Was ist ein „Modell"?
Das Modell = die trainierte Gewichtungstabelle = das „Gehirn"
- GPT-4, Claude Opus, Gemini, Grok 3 usw. = verschiedene Modelle mit verschiedenen Trainingsgrundlagen
- Alle sind LLMs (Large Language Models)

### Einfluss auf das Modell — drei Ebenen:

| Methode | Eingriff | Aufwand | Wirkung |
|---|---|---|---|
| **Von Grund auf trainieren** | Neues Modell | Strom wie New York 3 Monate | Eigenes Grundmodell |
| **Feintuning** | Gewichtungen leicht anpassen | Viel Strom + hunderte Seiten Material | Charakter-Anpassung |
| **System Prompting** | Verhaltensanweisung voranstellen | Minimal | Verhalten steuern (kein Charakter) |

> „Bleib beim System Prompting — da kannst du mehr bewirken als man denkt."

> „Wenn eine Firma sagt, wir haben eine eigene KI trainiert — dafür braucht es so viel Strom wie die Stadt New York in drei Monaten. Das machen nur OpenAI, Google, Anthropic etc."

### ✅ Learning
- System Prompting ist der realistische Hebel für alle Nicht-Tech-Firmen
- Feintuning: selten, teuer, nur für spezifische Zeichensprache / spezifisches Fachgebiet
- Eigenes Training: nur für die ganz Großen

## Glossar der Kernbegriffe

| Begriff | Bedeutung |
|---|---|
| **LLM** | Large Language Model — das trainierte „Gehirn" der KI |
| **Modell** | Eine spezifisch trainierte KI-Instanz (GPT-4o, Claude Opus, ...) |
| **Token** | Grob ein Wort; die kleinste Kommunikationseinheit der KI |
| **Transformer** | „Mund und Ohren" der KI — nimmt Prompt, gibt ein Token zurück |
| **GPT** | Generative Pre-trained Transformer — die Architektur hinter ChatGPT |
| **Next Word Prediction** | Das Grundprinzip: immer das wahrscheinlichste nächste Wort vorhersagen |
| **Prompt** | Die Eingabe an die KI (User Prompt) |
| **System Prompt** | Unsichtbarer Prompt, der vor dem User Prompt gesendet wird (3 Arten) |
| **Harness** | Die „Rüstung"/Karosserie um das Modell — stellt Tools bereit |
| **Tool Use** | Werkzeugaufruf durch die KI via Codewörter |
| **Agent** | KI + sinnvoller Tool-Einsatz = KI, die in der Welt handeln kann |
| **Reasoning / Thinking** | Stilles Nachdenken der KI vor der Antwort |
| **Feintuning** | Leichte Anpassung der Modell-Gewichtungen (aufwändig) |
| **Deterministisch** | Immer gleiche Ausgabe bei gleicher Eingabe (= Automatisierung) |
| **Context Engineering** | Die richtige Information zur richtigen Zeit an die KI bringen (Thema Session 2) |
| **Transformer-Durchlauf** | Ein Schritt: Prompt rein → ein Token raus |
